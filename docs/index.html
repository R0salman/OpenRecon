OpenRecon is a CLI-based, extensible reconnaissance and vulnerability assessment tool that brings together multiple scanning techniques in a single interface. It's built for security researchers, bug bounty hunters, penetration testers, and students in cybersecurity.

Requirements

You will need the following to use OpenRecon:

  1. Python 3.7+
  2. Nmap installed and in PATH
  3. Playwright for web application testing
  4. Internet Connection

Important: This tool is intended for authorized security testing only. Always obtain proper authorization before scanning any systems.

Overview

OpenRecon is a CLI-based, extensible reconnaissance and vulnerability assessment tool that brings together multiple scanning techniques in a single interface. It's built for security researchers, bug bounty hunters, penetration testers, and students in cybersecurity who want a practical and efficient way to identify surface-level threats and common vulnerabilities in networks and web applications.

I created OpenRecon as a learning project to better understand real-world web security, especially the OWASP Top 10 vulnerabilities. By building individual scanners for each vulnerability, I aimed to deepen my knowledge and gain hands-on experience while creating something useful for others exploring the same path in cybersecurity.

While OpenRecon currently includes four scanners, I'm actively committed to learning more and continuing to build and expand its capabilities as I grow in my cybersecurity journey.

OpenRecon provides comprehensive vulnerability assessment capabilities across multiple domains:

Network Reconnaissance

  • Port Scanning: Complete nmap integration with service detection
  • Host Discovery: ARP-based scanning for local networks
  • Ping Sweep: ICMP-based wide-range host detection

Web Application Security

  • XSS Scanner: Detects reflected, stored, and DOM-based Cross-Site Scripting
  • SQL Injection: Identifies potential SQLi vulnerabilities
  • CSRF Scanner: Checks for Cross-Site Request Forgery issues
  • GraphQL Scanner: Discovers exposed GraphQL endpoints
  • CMS Detection: Fingerprints WordPress, Joomla, and other CMS platforms

Information Gathering

  • Subdomain Enumeration: Uses crt.sh and DNS resolution with local caching
  • Payload Collection: Curated security testing payloads

Installation

Method 1: pip Installation (Recommended)

pip install openrecon

Note: If you encounter a "command not found" error after installation, the executable may not be in your system PATH.

Quick Fix (Immediate Use)
python -m openrecon --help
Permanent Fix (Windows)
  1. Copy the Scripts directory path from the warning message
  2. Press Win + R, type sysdm.cpl, press Enter
  3. Go to Advanced tab → Environment Variables
  4. Under User variables, select Path → Edit
  5. Click New and paste the Scripts directory path
  6. Click OK on all dialogs and restart your terminal

Method 2: Source Installation

Linux

Windows

Quick Start

Basic Usage

                        # Display help information
                        openrecon --help

                        # Show version
                        openrecon --version

                        # Network port scanning
                        openrecon scan --target example.com --flags "-T4 -F"

                        # Subdomain enumeration
                        openrecon enum --domain example.com

                        # Network discovery
                        openrecon disc --range 192.168.1.0/24 --ping

                        # Web vulnerability scanning
                        openrecon xss https://example.com
                        openrecon sqli https://example.com/search.php?q=test
                        openrecon csrf https://example.com/login

First-Time Setup Verification

                        openrecon --version

                        # Verify basic functionality
                        openrecon scan --target scanme.nmap.org --flags "-p 80,443"
                        openrecon enum --domain example.com

Usage Guide

Command Structure

                        openrecon [COMMAND] [ARGUMENTS] [OPTIONS]

Global Options

  • --version, -v, --ver: Show version information
  • --help: Display help information

Available Commands

  • scan: Network port scanning
  • enum: Subdomain enumeration
  • disc: Network discovery (ping/ARP)
  • sqli: SQL injection scanning
  • xss: Cross-site scripting scanning
  • csrf: CSRF vulnerability scanning
  • cms: CMS detection and fingerprinting
  • graphql: GraphQL endpoint scanning
  • util: Utility functions

Modules

scan - Port Scanning

This script automates network reconnaissance using Nmap. It simplifies scanning, handles input/output, and generates JSON results.

Examples
                    # Basic port scan
                    openrecon scan --target example.com --flags "-sS -T4"

                    # Fast scan of common ports
                    openrecon scan --target 192.168.1.1 --flags "-F"

                    # Comprehensive scan with service detection
                    openrecon scan --target scanme.nmap.org --flags "-sV -A -T4"

                    # Scan specific ports
                    openrecon scan --target example.com --flags "-p 22,80,443,8080"

                    # Stealth scan
                    openrecon scan --target example.com --flags "-sS -f -T2"
Required Arguments
  • --target, -t: Target IP, domain, or range
  • --flags, -f: Nmap flags for scanning
Common nmap flags
  • -sS: SYN scan (stealth)
  • -sV: Service version detection
  • -A: Aggressive scan
  • -T4: Aggressive timing
  • -F: Fast scan (top 100 ports)
  • -p-: Scan all 65535 ports
  • -f: Fragment packets

disc - Network Discovery

Performs ARP-based discovery within a specified IP range to find active devices on the network.

Examples
                    # Ping sweep
                    openrecon disc --range 192.168.1.0/24 --ping

                    # ARP discovery
                    openrecon disc --range 192.168.1.0/24 --arp

                    # Discovery with DNS
                    openrecon disc --range 10.0.0.0/24 --ping --dns --verbose

                    # Custom timing
                    openrecon disc --range 192.168.1.0/24 --ping --timeout 2.0 --threads 50 --retry 3
Arguments
  • --range, -r: IP range
  • --ping: Use ICMP ping
  • --arp: Use ARP
  • --timeout, --retry, --threads
  • --dns: Enable reverse lookups
  • --rate, --verbose

enum - Subdomain Enumeration

Enumerates subdomains using multiple sources and verifies them with DNS. Supports recursion, caching, and multi-threading.

Examples
                    openrecon enum --domain example.com
                    openrecon enum --domain example.com --recursive --depth 2
                    openrecon enum --domain example.com --threads 10 --timeout 30
                    openrecon enum --domain example.com --no-dns
                    openrecon enum --domain example.com --sources crt,alien,hacker
                    openrecon enum --domain example.com --output subdomains.txt --verbose
Arguments
  • --domain, -d (required)
  • --recursive, --depth, --no-dns
  • --cache: control and view options
  • --sources: choose data sources
  • --threads, --timeout, --output
Data Sources
  • crt.sh, AlienVault OTX, HackerTarget, BufferOver, ThreatCrowd, DNS brute-force

Web Application Security

xss - Cross-Site Scripting Scanner

Detects XSS vulnerabilities: reflected, stored, and DOM-based. Uses payload mutation, crawling, and headless browser testing.

Examples
                    openrecon xss https://example.com/search.php
                    openrecon xss https://example.com --type all
                    openrecon xss https://example.com --payloads custom_xss.json --threads 10
                    openrecon xss https://example.com --cookie "PHPSESSID=abc123"
                    openrecon xss https://example.com --crawl 2 --max-urls 20 --verbose
                    openrecon xss https://example.com --output xss_results --format json
Arguments
  • url (required)
  • --type: reflected, dom, stored, all
  • --threads, --crawl, --max-urls
  • --delay, --timeout, --verify-ssl
  • --output, --format, --verbose
  • --cookie, --user-agent, --proxy
  • --no-forms, --no-params, --payloads
  • --callback-url: blind XSS detection

Configuration

Configuration File

OpenRecon uses a configuration file located at ~/.openrecon/config.yaml.

                        # Default configuration
                        general:
                        timeout: 10
                        threads: 10
                        output_format: json
                        user_agent: "OpenRecon/1.0"

                        scanning:
                        nmap_default_flags: "-T4 -F"
                        max_ports: 65535
                        
                        web:
                        follow_redirects: true
                        verify_ssl: false
                        max_request_size: 1024
                        
                        cache:
                        enabled: true
                        ttl: 3600  # 1 hour
                        location: "~/.openrecon/cache"

Environment Variables

                        export OPENRECON_CONFIG="/path/to/config.yaml"
                        export OPENRECON_CACHE_DIR="/path/to/cache"
                        export OPENRECON_LOG_LEVEL="INFO"

Advanced Usage

Chaining Commands and Workflows

                        # Complete reconnaissance workflow
                        # 1. Discover subdomains
                        openrecon enum --domain example.com --output subdomains.txt

                        # 2. Port scan discovered hosts
                        cat subdomains.txt | while read subdomain; do
                            openrecon scan --target "$subdomain" --flags "-F" 
                        done

                        # 3. Web vulnerability testing
                        cat subdomains.txt | while read subdomain; do
                            openrecon xss "https://$subdomain" --output "xss_$subdomain"
                            openrecon sqli "https://$subdomain" --output "sqli_$subdomain"
                        done

                        # Network discovery and scanning
                        openrecon disc --range 192.168.1.0/24 --ping --output live_hosts.txt
                        # Then scan discovered hosts

Batch Processing Examples

                        # Multiple target XSS scanning
                        urls=("https://site1.com" "https://site2.com" "https://site3.com")
                        for url in "${urls[@]}"; do
                            openrecon xss "$url" --output "xss_$(basename $url)" --format json
                        done

                        # Comprehensive web app testing
                        openrecon xss https://example.com --type all --output xss_results
                        openrecon sqli https://example.com --type all --output sqli_results  
                        openrecon csrf https://example.com --output csrf_results
                        openrecon cms https://example.com --output cms_results

Integration with Other Tools

                        # Integration with httpx for live subdomain checking
                        openrecon enum --domain example.com | httpx -silent | tee live_subdomains.txt

                        # Pipe results to nuclei for additional vulnerability scanning
                        cat live_subdomains.txt | nuclei -t vulnerabilities/

                        # Use with subfinder for enhanced subdomain discovery
                        subfinder -d example.com -silent | sort -u > external_subs.txt
                        openrecon enum --domain example.com --output openrecon_subs.txt
                        cat external_subs.txt openrecon_subs.txt | sort -u > all_subdomains.txt

Troubleshooting

Common Issues

"Command not found" Error

Problem: OpenRecon command not recognized after installation.

Solution:

  • Use python -m openrecon instead of openrecon
  • Add Python Scripts directory to PATH (see installation section)
  • On Windows, restart terminal after PATH modification
Module-Specific Errors

Problem: "Domain is required" or similar argument errors.

Solution:

  • Check command syntax: openrecon enum --domain example.com (not --target)
  • Use correct argument names for each module
  • Run openrecon [command] --help for module-specific help
Nmap Integration Issues

Problem: "nmap: command not found" or nmap-related errors.

Solution:

  • Install nmap from nmap.org
  • Ensure nmap is in system PATH
  • Test with nmap --version
  • Use full path if needed: --flags "/usr/bin/nmap -sS"

Debug Mode and Logging

                        # Enable debug output (if implemented)
                        export OPENRECON_DEBUG=1
                        openrecon scan --target example.com --flags "-F"

                        # Verbose output for troubleshooting
                        openrecon enum --domain example.com --verbose
                        openrecon xss https://example.com --verbose

                        # Check module-specific help
                        openrecon scan --help
                        openrecon enum --help
                        openrecon xss --help
